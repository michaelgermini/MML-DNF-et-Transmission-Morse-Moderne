# 11.2 Déclarations : Éléments et attributs normalisés

## Catalogue complet des éléments MML

### Classification fonctionnelle

Les éléments MML sont classés selon leur fonction principale dans la structure documentaire :

#### 1. Structure et métadonnées
#### 2. Contenu textuel
#### 3. Médias et objets
#### 4. Liens et références
#### 5. Listes et énumérations
#### 6. Tableaux
#### 7. Formulaires
#### 8. Contenu spécialisé

## 1. Structure et métadonnées

### #META - Métadonnées documentaires

**Description** : Contient les informations générales sur le document

**Attributs** :
- `title` (string) : Titre du document
- `author` (string) : Auteur ou créateur
- `version` (string) : Version du document
- `created` (datetime) : Date de création
- `modified` (datetime) : Date de modification
- `language` (string) : Code langue (ISO 639-1)
- `license` (string) : Licence d'utilisation
- `description` (string) : Description courte
- `keywords` (string) : Mots-clés séparés par virgules

**Exemple** :
```
#META[title="Guide d'urgence"][author="Dr. Dubois"][language="fr"][version="1.0"]
```

### #SECTION - Section thématique

**Description** : Divise le document en sections logiques

**Attributs** :
- `id` (id) : Identifiant unique
- `class` (string) : Classes CSS
- `title` (string) : Titre de la section

**Contenu** : Éléments enfants (titres, paragraphes, etc.)

**Exemple** :
```
#SECTION[id="introduction"][title="Introduction"]
  #H2 Présentation
  #P Contenu de la section
```

### #DIV - Conteneur générique

**Description** : Conteneur sans sémantique spécifique

**Attributs** :
- `id` (id) : Identifiant unique
- `class` (string) : Classes CSS
- `style` (string) : Styles inline (déconseillé)

**Contenu** : Tous types d'éléments

### #H1 à #H6 - Titres hiérarchiques

**Description** : Titres de niveaux croissants d'importance

**Attributs communs** :
- `id` (id) : Identifiant unique
- `class` (string) : Classes CSS
- `level` (integer) : Niveau numérique explicite

**Règles hiérarchiques** :
- DOIT respecter l'ordre logique (pas de H3 directement après H1)
- PEUT sauter des niveaux si justifié
- DOIT être unique dans le document pour H1

## 2. Contenu textuel

### #P - Paragraphe

**Description** : Unité discursive de texte continu

**Attributs** :
- `id` (id) : Identifiant unique
- `class` (string) : Classes CSS
- `lang` (string) : Langue du paragraphe

**Contenu** : Texte avec éléments inline

### #BLOCKQUOTE - Citation longue

**Description** : Citation d'une source externe

**Attributs** :
- `cite` (uri) : Source de la citation
- `lang` (string) : Langue de la citation

**Contenu** : Texte de la citation

### #PRE - Texte préformaté

**Description** : Texte où l'espacement doit être préservé

**Attributs** :
- `id` (id) : Identifiant unique
- `class` (string) : Classes CSS

**Contenu** : Texte avec espaces et sauts de ligne significatifs

### Éléments inline

#### #STRONG - Emphase forte

**Description** : Texte d'importance ou d'urgence particulière

#### #EM - Emphase légère

**Description** : Accent ou stress sur le texte

#### #CODE - Fragment de code

**Description** : Code informatique ou commande

**Attributs** :
- `lang` (string) : Langage de programmation

#### #MARK - Texte marqué

**Description** : Texte surligné pour attirer l'attention

#### #Q - Citation courte

**Description** : Citation courte dans le texte

**Attributs** :
- `cite` (uri) : Source de la citation

#### #ABBR - Abréviation

**Description** : Abréviation avec expansion

**Attributs** :
- `title` (string) : Signification complète (obligatoire)

#### #TIME - Date/heure

**Description** : Date ou heure avec sémantique

**Attributs** :
- `datetime` (datetime) : Valeur machine (obligatoire)

## 3. Médias et objets

### #IMG - Image

**Description** : Référence à une image

**Attributs** :
- `src` (uri) : Source de l'image (obligatoire)
- `alt` (string) : Description textuelle (obligatoire)
- `width` (integer) : Largeur en pixels
- `height` (integer) : Hauteur en pixels
- `title` (string) : Info-bulle

### #VIDEO - Vidéo

**Description** : Contenu vidéo

**Attributs** :
- `src` (uri) : Source de la vidéo
- `controls` (boolean) : Afficher les contrôles
- `autoplay` (boolean) : Lecture automatique
- `loop` (boolean) : Lecture en boucle
- `width` (integer) : Largeur
- `height` (integer) : Hauteur

**Enfants** : Éléments #SOURCE pour sources multiples

### #AUDIO - Audio

**Description** : Contenu audio

**Attributs** : Similaires à #VIDEO mais pour l'audio

### #SOURCE - Source média

**Description** : Source alternative pour médias

**Attributs** :
- `src` (uri) : URL de la source
- `type` (string) : Type MIME

### #OBJECT - Objet générique

**Description** : Objet externe (PDF, etc.)

**Attributs** :
- `data` (uri) : Source de l'objet
- `type` (string) : Type MIME
- `width` (integer) : Largeur
- `height` (integer) : Hauteur

### #EMBED - Contenu intégré

**Description** : Contenu externe intégré

**Attributs** :
- `src` (uri) : Source du contenu
- `type` (string) : Type MIME
- `width` (integer) : Largeur
- `height` (integer) : Hauteur

## 4. Liens et références

### #LINK - Lien hypertexte

**Description** : Référence navigable

**Attributs** :
- `href` (uri) : Destination du lien (obligatoire)
- `title` (string) : Info-bulle
- `rel` (string) : Relation sémantique
- `target` (string) : Cible d'ouverture
- `lang` (string) : Langue de la destination

### #ANCHOR - Point d'ancrage

**Description** : Point de référence dans le document

**Attributs** :
- `name` (id) : Nom de l'ancre (obligatoire)
- `id` (id) : Identifiant alternatif

## 5. Listes et énumérations

### #UL - Liste non ordonnée

**Description** : Collection d'éléments sans ordre spécifique

**Attributs** :
- `id` (id) : Identifiant unique
- `class` (string) : Classes CSS

**Enfants** : Éléments #LI uniquement

### #OL - Liste ordonnée

**Description** : Collection d'éléments ordonnés

**Attributs** : Similaires à #UL plus :
- `type` (string) : Style de numérotation ("1", "A", "a", "I", "i")
- `start` (integer) : Numéro de départ

### #LI - Élément de liste

**Description** : Item individuel dans une liste

**Attributs** :
- `value` (integer) : Valeur spécifique (pour listes ordonnées)
- `id` (id) : Identifiant unique

**Contenu** : Texte avec éléments inline

### #DL - Liste de définitions

**Description** : Associations terme-définition

**Enfants** : Séquence alternée #DT/#DD

### #DT - Terme défini

**Description** : Terme à définir

### #DD - Définition

**Description** : Définition du terme précédent

## 6. Tableaux

### #TABLE - Tableau

**Description** : Données tabulaires structurées

**Attributs** :
- `caption` (string) : Titre du tableau
- `summary` (string) : Description du contenu
- `id` (id) : Identifiant unique

**Enfants** : #THEAD, #TBODY, #TFOOT

### #THEAD - En-tête de tableau

**Description** : Section d'en-tête

**Enfants** : Éléments #TR

### #TBODY - Corps du tableau

**Description** : Section de données principales

### #TFOOT - Pied de tableau

**Description** : Section de synthèse

### #TR - Ligne de tableau

**Description** : Ligne horizontale

**Enfants** : Éléments #TH ou #TD

### #TH - Cellule d'en-tête

**Description** : Cellule d'en-tête

**Attributs** :
- `scope` (string) : Portée ("row", "col", "rowgroup", "colgroup")
- `colspan` (integer) : Fusion horizontale
- `rowspan` (integer) : Fusion verticale

### #TD - Cellule de données

**Description** : Cellule de données normales

**Attributs** : Similaires à #TH

## 7. Formulaires

### #FORM - Formulaire

**Description** : Conteneur de champs interactifs

**Attributs** :
- `action` (uri) : URL de soumission (obligatoire)
- `method` (string) : Méthode HTTP ("GET", "POST")
- `enctype` (string) : Type d'encodage
- `id` (id) : Identifiant unique

**Enfants** : Champs de formulaire

### #INPUT - Champ de saisie

**Description** : Champ de saisie utilisateur

**Attributs** :
- `type` (string) : Type de champ ("text", "password", "email", etc.)
- `name` (string) : Nom du champ (obligatoire)
- `value` (string) : Valeur par défaut
- `placeholder` (string) : Texte d'aide
- `required` (boolean) : Champ obligatoire
- `disabled` (boolean) : Champ désactivé
- `readonly` (boolean) : Lecture seule
- `maxlength` (integer) : Longueur maximale

### #TEXTAREA - Zone de texte multi-ligne

**Description** : Champ de texte étendu

**Attributs** : Similaires à #INPUT plus :
- `rows` (integer) : Nombre de lignes
- `cols` (integer) : Nombre de colonnes
- `wrap` (string) : Mode de retour à la ligne

### #SELECT - Liste déroulante

**Description** : Sélection dans une liste

**Attributs** :
- `name` (string) : Nom du champ (obligatoire)
- `multiple` (boolean) : Sélection multiple
- `size` (integer) : Nombre d'options visibles

**Enfants** : Éléments #OPTION

### #OPTION - Option de sélection

**Description** : Option individuelle

**Attributs** :
- `value` (string) : Valeur soumise
- `selected` (boolean) : Pré-sélectionnée

### #BUTTON - Bouton

**Description** : Bouton interactif

**Attributs** :
- `type` (string) : Type de bouton ("submit", "reset", "button")
- `name` (string) : Nom du bouton
- `value` (string) : Valeur soumise
- `disabled` (boolean) : Désactivé

### #LABEL - Étiquette

**Description** : Texte descriptif pour un champ

**Attributs** :
- `for` (id) : ID du champ associé

### #FIELDSET - Groupe de champs

**Description** : Regroupement logique de champs

**Attributs** :
- `disabled` (boolean) : Groupe désactivé

**Enfants** : Attribut `legend` implicite, puis champs

## 8. Contenu spécialisé

### #ADDRESS - Informations de contact

**Description** : Coordonnées et informations de contact

### #CITE - Référence bibliographique

**Description** : Citation d'une œuvre

### #DETAILS - Contenu pliable

**Description** : Contenu masquable/affichable

**Enfants** : #SUMMARY puis contenu

### #SUMMARY - Résumé pliable

**Description** : En-tête du contenu pliable

### #PROGRESS - Indicateur de progression

**Description** : Barre de progression

**Attributs** :
- `value` (float) : Valeur actuelle
- `max` (float) : Valeur maximale

### #METER - Jauge

**Description** : Indicateur de mesure

**Attributs** :
- `value` (float) : Valeur actuelle
- `min` (float) : Valeur minimale
- `max` (float) : Valeur maximale
- `low` (float) : Seuil bas
- `high` (float) : Seuil haut
- `optimum` (float) : Valeur optimale

## Attributs universels

### Attributs applicables à tous les éléments

#### `id` (id)
**Description** : Identifiant unique dans le document
**Usage** : Références, styles, scripting

#### `class` (string)
**Description** : Classes CSS séparées par espaces
**Usage** : Stylage, sélection programmatique

#### `title` (string)
**Description** : Information complémentaire
**Usage** : Info-bulles, accessibilité

#### `lang` (string)
**Description** : Langue du contenu (code ISO 639-1)
**Usage** : Internationalisation, synthèse vocale

#### `dir` (string)
**Description** : Direction du texte ("ltr", "rtl")
**Usage** : Langues bidirectionnelles

#### `translate` (boolean)
**Description** : Élément traduisible
**Usage** : Outils de traduction

## Attributs de sécurité

### `hash` (string)
**Description** : Empreinte cryptographique pour l'intégrité
**Format** : Hexadécimal, algorithme dépendant
**Usage** : Vérification d'intégrité

### `signature` (string)
**Description** : Signature numérique pour l'authentification
**Format** : Base64, algorithme dépendant
**Usage** : Vérification d'origine

### `encrypted` (boolean)
**Description** : Contenu chiffré
**Usage** : Indicateur de chiffrement

### `key_id` (string)
**Description** : Identifiant de clé de chiffrement
**Usage** : Résolution de clé

## Règles de validation par élément

### Règles obligatoires

- `#IMG` DOIT avoir `alt`
- `#LINK` DOIT avoir `href`
- `#INPUT` DOIT avoir `name` (sauf boutons)
- `#OPTION` PEUT avoir `value`
- `#TIME` DOIT avoir `datetime`

### Règles de cohérence

- `#LI` NE DOIT PAS apparaître hors de #UL/#OL/#DL
- `#DT` et `#DD` DOIVENT alterner dans #DL
- `#TH` NE DOIT PAS apparaître dans #TBODY
- `#OPTION` NE DOIT PAS apparaître hors de #SELECT

### Règles sémantiques

- `#H1` DEVRAIT être unique par document
- `#SECTION` DEVRAIT contenir au moins un titre
- `#FORM` DEVRAIT avoir au moins un champ soumis
- `#TABLE` DEVRAIT avoir `caption` pour l'accessibilité

---

**Note** : Cette spécification évolue. Les extensions sont marquées par le préfixe `X` pour les éléments expérimentaux.
